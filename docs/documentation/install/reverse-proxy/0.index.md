### Why doesn't Audiobookshelf have built-in SSL/HTTPS support?

Audiobookshelf is designed to serve unencrypted HTTP. To use HTTPS, you must use a reverse proxy like above to handle the SSL/TLS decryption.

This is a design choice to keep Audiobookshelf secure by having encryption and certificate management handled by dedicated, proven web servers. Additionally, using a reverse proxy allows you to easily manage SSL certificates for multiple self-hosted services from a single, centralized entry point. It also makes sure that a critical reverse proxy only needs to be updated at one point, instead of every single application.
